## The Pipeline

Most engineers build systems for corporations.  
I build them for farmers who never had access to one.

My day job is designing ETL pipelines, migrating databases, and making APIs talk to each other cleanly.  
My side work is putting that same infrastructure behind people who need it most.

## How I Think About Work

Data engineering is plumbing.  
Nobody notices when it works. Everyone notices when it breaks.

Good pipelines are invisible. Good products are inevitable.  
I try to build both.

---

## Customer Success + Data

Most data engineers stop at the pipeline.  
I sit downstream — with customer success and query resolution — so I understand what the data is actually being used for.  
That context changes how I design systems.
